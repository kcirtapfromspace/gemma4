## Winner Candidates (Ranked by Effort for Working Gemma 4 MTP by 2026-05-18)

### Tier 1: Ready Today (0–2 days integration)
1. **Hugging Face Transformers** ⭐ **RECOMMENDED**
   - Day-0 MTP support + universal assisted generation.
   - Mature, no critical bugs. Safetensors format natively supported.
   - Baseline for all other runtimes.

2. **vLLM** ⭐ **STRONG SECOND**
   - Spec-decode production-ready; Google engineer PR for Gemma 4 MTP merged May 2026.
   - Latest v0.7+ includes zero-bubble async + GPU ngram. Serving-focused (batch inference).
   - ~1 day to integrate, test, hardened for production use.

3. **LiteRT-LM** ⭐ **MOBILE LEADER**
   - Ships MTP heads. Kotlin/Java APIs stable. CLI flag ready.
   - iOS Swift API in-development (not production-safe for hackathon).
   - ~2 days to harden Kotlin surface on device.

### Tier 2: Medium Effort (3–7 days)
4. **MLX** (Apple Silicon)
   - Day-0 official support; assisted generation native. Safetensors support mature.
   - Caveat: MoE models (26B-A4B) need batch-size tuning; single-batch suboptimal.
   - ~3 days for single-batch inference tuning or multi-batch harness.

5. **SGLang** (Advanced serving)
   - Spec-decode mature (EAGLE/MTP/draft variants).
   - Auto-tuning via bench_speculative.py. No reported Gemma 4 blockers.
   - ~4 days for full serving harness (more if custom inference loop needed).

### Tier 3: Not Ready by Deadline (8+ days)
6. **Ollama** (Desktop/CLI)
   - Speculative decoding feature PR (#8134) not yet merged to stable.
   - Gemma 4 tool calling broken; Flash Attention hangs on 31B.
   - ~7+ days to unblock + stabilize.

7. **MLC-LLM** (Jetson/embedded native)
   - Gemma 4 architecture (PLE, multimodal towers, parameter nesting) requires TVM refactor.
   - No roadmap; **not viable by 2026-05-18** without Google/Meta sponsorship.
   - ~21 days to upstream-ready fix; 7 TVM patches in local repo insufficient.

8. **llama.cpp** (Bleeding-edge)
   - MTP heads support PR (#22673) in progress, not merged.
   - GGUF quantization loses MTP heads; custom fork required.
   - ~10+ days to merge + stabilize GGUF export + test.

## Recommendation for Hackathon

**Go with Tier 1: Hugging Face Transformers (primary) + vLLM (serving variant).**

- **Primary path (simplest):** Transformers + Python inference script. 1 day integration; zero risk.
- **Optional variant (production serving):** vLLM + `--speculative-model` flag. 1 day extra for serving harness; mature spec-decode.
- **Mobile path (if needed):** LiteRT-LM (Kotlin) or MLX (Apple Silicon) as secondary. Both viable in parallel within 2 days.

Avoid MLC-LLM and llama.cpp: both hit architectural blockers that exceed the 2-week budget. Focus on shipping a hardened, measurable improvement with a stable runtime.
